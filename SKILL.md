## 核心原则（最优先，必须始终遵循）

> **技术是手段，人是核心。**
> 我们无法教计算机学习人类无法学习的东西。AI能发现海量数据中的隐藏模式，但无法替代人类对消费者行为动机的深层理解。营销人员的判断力在复杂情境下仍不可替代。机器做标准化高频交互，人做情感连接和复杂决策。

> **消费者有"选择性关注"特点。**
> 人类天生倾向于过滤无关信息，只关注相关事物。过多的产品选择和商业信息会让消费者分心、难以决策。AI应代替人脑做信息过滤，而非强迫消费者处理所有信息。

> **个性化（Personalized）与定制化（Customized）有本质区别。**
> **个性化（Personalized）**：AI预测消费者需求并主动推送，技术主导、消费者被动接受推荐结果。例如：算法推荐、预测性营销。
> **定制化（Customized）**：消费者参与决策过程，控制权在消费者。例如：让消费者选择原料/配置/包装。两者不能混用——选了预测性营销手段不代表实现了定制化。

> **营销5.0是范式转移，不是降本增效工具。**
> 营销4.0是"数字化营销"（提升效率），营销5.0是"数智化营销"（改变竞争优势的根本路径）。本质区别：数字化转型是过程，数智化是目的。

> **"营销的使命永远是改善人类的生活，促进公众利益的实现。"——菲利普·科特勒**

## 核心架构（营销5.0的真实层次）

### 正确的五层结构（非平列，是层次）

```
         ┌─────────────────────────────────┐
         │  对话式/增强型营销（次级支撑）      │
         │  NLP机器人 · 智能客服 · 一线赋能   │
         └──────────────┬──────────────────┘
                        ↕
┌───────────────────────┼─────────────────────────────────────────┐
│        核心技术层      │                                         │
│  预测性营销 ─────────┼──────── 情境化营销 ───────── 增强现实营销 │
│  AI预测行为 │         │  全渠道无缝 │                AR/VR体验 │
│  新品验证   │         │  场景触发   │                沉浸展示  │
└─────────────┼───────────┴──────────────┼─────────────────────────┘
              ↓                              ↓
┌──────────────────────────┐  ┌──────────────────────────────────┐
│       基础层              │  │           基础层                   │
│  数据驱动型营销           │  │  敏捷营销                         │
│  (数据生态系统)           │  │  (快速迭代 · 实时响应)            │
│  采集/处理/应用三层      │  │  Scrum · Kanban · 设计思维        │
└──────────────────────────┘  └──────────────────────────────────┘
```

**关系逻辑：**
- 基础层是**前提条件**：没有数据基础，预测/情境/增强现实全部失效
- 敏捷营销是**执行保障**：快速测试 → 迭代优化
- 核心技术层是**主攻手段**：三者可以组合使用
- 对话式是**辅助支撑**：增强其他手段的效果

**⚠️ 包容性设计原则（营销5.0特有）：**
- 数字技术应用应兼顾不同数字成熟度用户，不能抛弃低数字成熟度群体
- 技术选择应考虑社会包容性，不能加剧数字鸿沟
- 企业社会责任（CSR）与营销5.0技术手段应结合（如普惠性数字服务）

## 评估模式：如何诊断策划案

### 评估顺序（逐层深挖，框架层问题优先）

**🔴 第一层：前提条件检查**

检查策划案是否具备营销5.0的**基础条件**：

1. 三大社会背景是否覆盖？
   - 消费者代沟：[有/无] → 若无，严重扣分
   - 贫富两极化：[有/无] → 若无，提醒
   - 数字鸿沟：[有/无] → 若无，提醒

2. 数据生态系统是否存在？
   - [有基本数据/数据薄弱/几乎无数据]
   - 若"数据薄弱"或"几乎无数据"，策划案的预测性/情境化营销技术选择无效

3. 是否在营销4.0基础之上谈5.0？
   - [有/无] → 若无，说明策划案跳过了必要的数字化基础

4. 是否回答了"如果不改变，结果是什么"？
   - [有/无] → 若无，说明策划案缺乏变革紧迫性论证

**🔴 第二层：技术手段选择诊断**

1. 技术手段是否有**明确匹配的业务问题**？
   - [有/无] → 若无，这是框架层错误，需要重新选择
   - "我们想做AI营销"不是业务问题；"客户流失率高但原因不明"才是业务问题

2. 技术手段的**前置条件**是否具备？
   - 预测性营销需要6-12个月历史数据
   - 情境化营销需要IoT传感器/数字化触点基础
   - 增强现实营销需要足够数字成熟度用户群体

3. 技术手段之间是否有**协同逻辑**？
   - 不应堆砌所有技术；选择1-2个主攻+1-2个辅助即可

**🟡 第三层：人机分工诊断**

1. 是否有**人机分工矩阵**？
   - 策划案是否明确哪些由机器做、哪些由人做？
   - 若全篇没有出现"人工"的场景，说明策划案可能过度AI化

2. 人工保留的场景是否有**真正的情感价值**？
   - 高净值客户关系、复杂谈判、品牌价值观传递等场景不可被AI替代

**🟡 第四层：全渠道闭环诊断**

1. 5A旅程是否覆盖完整（Aware→Appeal→Ask→Act→Advocate）？
   - 常见断点：Ask（无互动机制）和Advocate（无复购/推荐设计）

**5A旅程断点诊断清单（逐项检查）：**
| 环节 | 诊断问题 | 通过✓/不通过✗ |
|------|---------|-------------|
| Aware认知 | 获客渠道是否多元化？内容是否匹配目标世代的媒体习惯？ | [___] |
| Appeal吸引 | 价值主张是否与竞品形成差异化？是否回答"为什么选你不选别人"？ | [___] |
| Ask询问 | 是否有互动/咨询入口？客户疑问是否有人响应？ | [___] |
| Act行动 | 转化路径是否顺畅？是否有导致用户流失的断点（如注册流程复杂）？ | [___] |
| Advocate倡导 | 是否有复购机制？是否有推荐激励（如老带新/口碑传播）？ | [___] |

2. 情境化营销的**触发机制**是否存在？
   - 策划案是否设计了具体的场景触发点？

3. 线上线下数据是否打通？

**🟡 第四层补充：增长逻辑诊断（营销5.0特有）**

策划案的每个策略是否都能回答"这个投入如何驱动业务增长"？
| 策略/投入 | 预期的增长路径 | 可量化指标 | 逻辑是否成立 |
|----------|--------------|----------|------------|
| [___] | [___] | [___] | [✓/✗] |

**⚠️ 如果策划案是"花钱计划"而不是"增长方案"：** 这是框架层问题，必须打回重做

**🟢 第五层：执行可行性诊断**

1. 敏捷验证机制是否存在？
   - 策划案是否设计了MVP测试方案？
   - 是否有明确的成功/失败指标？

2. KPI是否可量化、可追踪？
3. 预算与策略是否匹配？

### 评估输出格式

```
【评估结论】

【前提条件评估】
- 三大背景覆盖：[完整/部分/缺失]
- 数据基础：[充分/薄弱/缺失]
- 4.0基础：[有/无]

【核心问题（最多3个，按严重程度）】
1. [框架层🔴/策略层🟡/执行层🟢]：[问题描述]
   → 严重程度：[🔴严重/🟡中等/🟢轻微]
   → 修改建议：[具体表述]

【营销5.0味评分】
- 前提条件完整性：X/10
- 技术手段匹配度：X/10（技术选择是否有明确业务问题支撑）
- 人机分工合理性：X/10
- 全渠道闭环性：X/10
- 敏捷验证机制：X/10
- 综合评分：X/50

【通过测试】
1. 三大社会背景是否全部覆盖？✓/✗
2. 技术手段选择是否有明确匹配的业务问题？✓/✗
3. 人机分工是否有明确的分工矩阵？✓/✗
4. 是否有敏捷验证回路和可量化KPI？✓/✗
5. 策划案是否回答了"不改变模式的结果是什么"？✓/✗

若任一答案为✗：策划案未达到营销5.0基本标准
```

## 修改模式：如何改策划方案

### 核心原则
**先框架后细节，先匹配后文案。**
文案是最后才动的东西。在框架和策略确认之前，不花时间改措辞。

### 三级修改法

**🔴 框架层修改（可能需要推翻重做）**

以下任一情况，必须框架层重做：

1. **背景分析缺失或不准确**
   - 典型错误：面向"追求品质的消费者"，未区分世代
   - → 重做：拆分为婴儿潮/X世代 vs Y/Z世代，用不同渠道+不同语言

2. **技术手段选择与业务问题不匹配**
   - 典型错误：写了"引入AI营销"但没有说清楚要解决AI能解决的哪个具体问题
   - → 重做：先定义业务问题，再匹配技术手段

3. **数据基础不具备的情况下选择了需要数据的技术**
   - 典型错误：选了预测性营销但没有6-12个月历史数据
   - → 重做：先补充数据建设路径，或者选择不依赖历史数据的技术

4. **策划案没有回答增长逻辑**
   - 典型错误：策划案是一个"花钱计划"而不是"增长方案"
   - → 重做：每个策略都必须回答"这个投入如何驱动业务增长"

**🟡 策略层修改（局部调整）**

1. 全渠道有断点 → 补充打通方案 + 情境触发点设计
2. 数据基础薄弱但技术选择合理 → 补充数据建设阶段路径（Phase 1/2/3）
3. KPI 不具体或不可衡量 → 重新定义KPI（必须是可量化的）
4. 5A旅程有断点 → 补充缺失环节的机制设计

**🟢 执行层修改（细节优化）**
- 文案/创意建议（确认框架后进行）
- 预算细节
- 时间排期

### 典型修改案例

**框架层修改：**
> 原文：面向"追求品质的消费者"
> 问题：未识别世代差异，"品质"在不同世代的定义完全不同
> 修改：拆分为"婴儿潮/X世代（高购买力，品质导向，线下信任渠道）"和"Y/Z世代（数字原住民，价值观驱动，线上种草渠道）"，分别制定触达策略和价值主张

**框架层修改：**
> 原文：引入AI营销提升客户体验
> 问题：AI营销是手段，不是目的；没有说清楚解决什么具体问题
> 修改：重新定义业务问题"客户流失率比行业均值高15%，原因不明" → 匹配预测性营销（识别流失信号）→ 补充数据基础要求

**策略层修改：**
> 原文：建立会员体系提升复购
> 问题：会员体系是手段，解决的是"复购"问题，但复购低的根本原因未诊断
> 修改：补充诊断"复购低的根本原因是什么？（流失预警缺失/场景触发不够/情感连接弱）" → 再选择对应的技术手段

## 谋划模式：如何出谋划策给方向

### 使用说明
用户描述公司/品牌面临的营销困境 → AI用营销5.0框架诊断 → 给出最大杠杆破局方向

### 谋划步骤

**步骤1：市场背景图与数字化成熟度自评**

先做三维度自评（每项1-10分）：
1. 消费者数字化转型推动力：___分
2. 企业数字化经营能力：___分
3. 数字化领导力水平：___分

自评<5分的维度 → 策划时必须专项加强

判断行业所处营销阶段（1.0-5.0）：
- 以产品功能差异为主？→ 营销1.0阶段
- 以品牌情感诉求为主？→ 营销2.0阶段
- 以价值观/社会责任为核心？→ 营销3.0阶段
- 以数字社群/共创为主？→ 营销4.0阶段
- 以类人技术（AI/IoT/NLP/AR）驱动增长为主？→ 营销5.0阶段

**主要竞争对手技术应用水平评估：**
| 竞品 | 数据资产 | AI/预测应用 | 数字化程度 | 技术差距 |
|------|---------|------------|----------|---------|
| 竞品A | [有/无] | [有/无] | [高/中/低] | [___] |
| 竞品B | [有/无] | [有/无] | [高/中/低] | [___] |
| 本企业 | [有/无] | [有/无] | [高/中/低] | [定位：___] |

**步骤2：识别核心痛点（对标技术手段）**

| 痛点（你公司的实际问题） | 匹配手段 | 匹配条件 |
|----------------------|---------|---------|
| 不知道客户下一步要什么，新品失败率高 | 预测性营销 | 需要6-12个月历史数据 |
| 渠道割裂，体验不一致 | 情境化营销 | 需要IoT/数字化触点 |
| 客服成本高/响应慢 | 对话式营销 | 有足够咨询量 |
| 产品难展示/体验不足 | 增强现实营销 | 目标用户数字成熟度高 |
| 市场反应慢/窗口期短 | 敏捷营销 | 组织能接受快速迭代 |
| 不知道该选哪个手段 | 先做自评和数据分析 | |

**⚠️ 注意：** 一个手段对应一个问题；没有明确问题的技术手段不应选择

**步骤3：找最大杠杆破局点**

判断标准（满足3条以上才选）：
1. **技术可行性高** → 现有数据/技术能支撑，或者3-6个月内能建立
2. **投资回报快** → 能在2个迭代周期（4-8周）内看到可量化效果
3. **竞争差异化强** → 竞品尚未布局或做得很差
4. **撬动其他环节** → 能带动策划案其他环节的提升
5. **组织适配性** → 企业是否具备相应组织能力（跨部门协作、数据素养、变革意愿）

### 谋划输出格式

```
【破局方向】

核心破局点（只选1个，不要贪多）：
做什么：[具体行动]
解决什么问题：[业务痛点/诊断结论]
预期效果：[可量化的结果，如"流失率降低X%"]
选择了该手段而非其他的理由：[___]

三大背景影响评估：
- 消费者代沟：[影响显著/影响较小，原因：___]
- 贫富两极化：[影响显著/影响较小，原因：___]
- 数字鸿沟：[影响显著/影响较小，原因：___]

配套动作（2-3个，为核心手段服务）：
1. [动作] → [如何辅助核心手段]
2. [动作] → [如何辅助核心手段]
3. [动作] → [如何辅助核心手段]

数据基础优先建设（核心手段落地前必须先有）：
- 阶段1（0-6月）：[___]
- 阶段2（6-12月）：[___]

里程碑验证节点：
- 节点1（X周后）：验证[某KPI]是否达标 → 达标则[扩大投入/复制]，不达标则[调整方案/暂停]
- 节点2（X周后）：[___]

营销5.0基础条件自评：
- 数据生态系统：[已建立/部分建立/待建立]
- 敏捷迭代能力：[有/待建设]
- 一线数字化工具：[有/待建设]
```

## 边界条件（营销5.0不适用的场景）

| 场景 | 为什么不适用 | 建议替代方案 |
|------|------------|------------|
| 目标用户数字化成熟度极低（如老年群体为主） | 情境/AR/对话式营销效果差，数字鸿沟无法弥合 | 回归营销3.0打法，专注人文价值/关系营销 |
| 产品没有数据触点（B2B工业品部分品类） | 无法建立数据基础，预测营销是空中楼阁 | 关系营销+内容营销+人工销售赋能优先 |
| 企业零数据资产 + 无数据建设预算 | 五种技术手段均无法真正落地 | 先做数据基础设施规划，3-6个月内不上5.0技术 |
| 强监管行业（医疗/金融合规限制） | AI/数据应用受合规限制，对话式/预测营销受限 | 人工服务为主，辅助轻量数字化（内容/社群） |
| 营销1.0阶段企业强行上全套5.0 | 组织能力不匹配，技术是浪费 | 先建立营销4.0基础（私域/数据/社群），再谈5.0 |

**策划案禁区（违反任一条，评分直接扣10分）：**
- ❌ 只谈技术不谈数据 → 技术是空中楼阁
- ❌ 全盘AI化忽视人工 → 情感连接不可替代，一线人员赋能才是正道
- ❌ 跳过敏捷验证直接大规模押注 → 快速测试是营销5.0核心原则，不验证就投入是赌博
- ❌ 用同一套策略打所有世代 → 代沟被忽视是策划案框架层的典型失败
- ❌ 策划案没有增长逻辑 → 花多少钱不是重点，花了带来多少增长才是重点
- ❌ 跳过4.0基础直接上5.0技术 → 营销4.0的数字化基础是5.0的前提

## 附录：营销1.0-5.0核心对照

| 阶段 | 核心 | 主导逻辑 | 典型企业/打法 |
|------|------|---------|------------|
| 营销1.0 | 产品中心 | 功能差异化卖点 | 工业化早期，功能竞争 |
| 营销2.0 | 消费者导向 | 品牌情感诉求 | 宝洁/可口可乐，品牌形象 |
| 营销3.0 | 人文精神 | 价值观共鸣/CSR | Patagonia/星巴克，社会责任 |
| 营销4.0 | 数字化 | 大数据/社群/共创 | 小米/完美日记，私域运营 |
| 营销5.0 | 类人技术 | AI+IoT+AR+NLP+敏捷 | 领先品牌转型，数智化营销 |

**本质区别：**
- 营销4.0 = 数字化（过程，效率提升）
- 营销5.0 = 数智化（目的，竞争优势的根本转移）
